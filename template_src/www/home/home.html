Google's material design is a visual style for mobile user interfaces. Aerophane is a template that embraces material design ideas to create an ultra light starting point for hybrid mobile web development.

Aerophane strives to have a small footprint, run blazingly fast, and be pleasant to modify while also implementing the most common features used in modern mobile UIs.

Section Header H3

Aerophane does not use copious classes to connect styles to HTML elements. Elements acquire their styling based on their position in the document. For example, the <article> tag is the basic container, a <section>` element inside an article is displayed in a "card" style, and a <header>` inside a section card looks like the blue space above. Check the source code of the exmaples for specifics.

Furthermore, HTML is not indented. HTML is document markup, not logical code. Combined with sparse class attributes and minimal HTML*, the UI code should be nearly as readable as a markdown document.

Once you initialize the Aerophane object, you have access to a few other JavaScript utility functions including:

forEachElement

The JavaScript forEach array method doesn't work on element collections, so I find his handy to avoid writing for loops. 

aero.forEachElement(elementCollection, elementFunction);

classname

Another utility function that Aerophane adds that would be completely unecessary if you are using jQuery is a method to work with class attributes. 

aero.classname.add(element, classname);
aero.classname.remove(element, classname);

*Minimal HTML

Only use as much HTML as needed. If an element can be removed, it should be. If an element has an optional end tag, then don't use the end tag unless there is good reason.

Dialog

<div> elements with the class "dialog" that are a direct child of the body tag will be not displayed by default. They can be shown like this: 

aero.showDialog(element);

Touch the background to close.
